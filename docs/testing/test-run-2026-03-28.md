## Failures Detail

### FAIL [OB-05]: Stub state doesn't persist across reloads
- **Severity:** Low (stub limitation, not app bug)
- **Expected:** setFirstRunCompleted(true) + reload should skip onboarding
- **Actual:** Onboarding appeared again because reload creates new stub instance with default state
- **Impact:** Cannot test "returning user" flow without completing onboarding each time
- **Fix:** Enhance stub to use sessionStorage for state persistence across reloads

### FAIL [KB-01]: `?` key opens search palette instead of shortcuts overlay
- **Severity:** Medium (UX issue)
- **Expected:** Shortcuts overlay with keyboard shortcut reference
- **Actual:** Search readings palette opened ("Search readings by title, author, source...")
- **Impact:** Users pressing `?` for help get search instead of keyboard reference
- **Fix:** Verify `?` handler — may need separate overlay or the shortcuts overlay may be the Hotkey Map settings page, not a modal

### FAIL [ERR-04]: update-available event has no visible UI response
- **Severity:** Medium (missing feature)
- **Expected:** Update notification banner or toast
- **Actual:** Event emitted to console but no visible notification
- **Impact:** Users won't know an update is available via IPC event
- **Fix:** Add event listener for `update-available` that shows a toast or banner

### FAIL [ERR-05]: cloud-auth-required event has no visible UI response
- **Severity:** Low (edge case)
- **Expected:** Auth prompt or re-login dialog
- **Actual:** Event emitted but no visible response
- **Impact:** Users won't be prompted to re-authenticate when cloud token expires
- **Fix:** Add event listener for `cloud-auth-required` that navigates to Cloud Sync settings or shows toast

### FAIL [MODE-04]: Shift+Space doesn't cycle from Narrate back to Focus
- **Severity:** Medium (broken cycle)
- **Expected:** Narrate → Focus (completing the Focus→Flow→Narrate→Focus cycle)
- **Actual:** Remained in Narrate mode (Shift+Space may have paused narration instead)
- **Impact:** Users can't cycle through all 3 modes — must Escape and manually re-enter Focus
- **Fix:** Verify Shift+Space handler in Narrate mode — may need to stop narration first before cycling

## Observations

### Systemic Issues
1. **Stub state persistence:** The biggest limitation is that stub state doesn't survive page reloads. This means any test requiring reload (OB-05, state verification) will fail. Consider using sessionStorage in the stub.

2. **G-sequence timing:** The two-keypress G-sequences (e.g., G→F for Favorites) don't reliably work via automation. The timing between keypresses may need tuning, or the implementation may require precise focus state.

3. **Browser shortcut conflicts:** Some shortcuts (Tab, Ctrl+,) conflict with Chrome browser defaults. These work correctly in the Electron app but not in the browser test harness.

4. **Short stub content:** The Meditations sample has only ~100 words across 3 short "books", all fitting on one page. This prevents testing pagination, chapter jumping, and long-form reading mode behavior.

### Positive Findings
1. **All 4 reading modes work:** Focus (RSVP), Flow (scroll cursor), Narrate (TTS), and Page (default) all render and function correctly.
2. **Mode cycling works:** Shift+Space successfully cycles Focus→Flow→Narrate (but not back to Focus).
3. **Theme switching is flawless:** All 5 themes (blurby, dark, light, eink, system) apply correctly with no CSS artifacts.
4. **Settings pages are comprehensive:** All 7 sub-pages render correctly with full controls.
5. **Command palette is excellent:** Fuzzy search works well, showing categorized results (ACTION, SETTING, SHORTCUT).
6. **Error handling works:** watcher-error event produces a proper toast notification.
7. **EPUB rendering via foliate-js works in browser:** The stub's base64 EPUB loads and renders correctly through the foliate-js reader.
8. **ORP highlighting is visually clear:** The optimal recognition point character is highlighted in gold in Focus mode.
9. **Bottom bar is well-organized:** WPM control, font size, mode buttons, and chapter dropdown all visible and functional.
10. **Onboarding flow is polished:** Welcome screen with Get Started/Skip tour options, 3-step tooltip tour works smoothly.
