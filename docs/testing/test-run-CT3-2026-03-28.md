## Observations

1. **G-sequences are the primary shortcut issue.** All G-sequence shortcuts (8 of them) failed consistently via automation. This is almost certainly a timing issue with the browser automation tool rather than an app bug — the single-key shortcuts (`?`, `;`, `/`, `Tab`, `s`) all work perfectly. **Recommendation:** Verify G-sequences manually in Electron app.

2. **Keyboard shortcuts overlay vs checklist mismatch.** The actual shortcut mappings in the app differ from the checklist in several places:
   - App: `G+S` = Go to starred/favorites; Checklist: `gf` = favorites, `gs` = stats
   - App: `G+I` = Go to inbox/unread; Checklist: no `gi` mapping
   - App: `G+G` = Jump to top; Checklist: no `gg` mapping
   - **Action needed:** Update checklist to match actual implementation.

3. **Ctrl+, shortcut conflict.** This opened the Reading Queue sidebar instead of Settings. This could be a Chrome browser-level conflict or a keybinding issue. The command palette `Ctrl+K` correctly shows "Open Settings" as an option, and Settings is accessible via menu. Low impact since alternative paths exist.

4. **Narrate mode Kokoro toggle.** The engine switch (Web Speech ↔ Kokoro) toggle was not visible during narration playback. This may be intentional for the browser stub (Kokoro not available in browser), but should be verified in Electron.

5. **All core reading modes work flawlessly.** Page, Focus, Flow, and Narrate modes all function correctly. Mode cycling via Shift+Space works perfectly in the Focus → Flow → Narrate → Focus loop. Word tracking, ORP highlights, underline cursors, and speed controls all verified.

6. **Toast system works well.** Error toasts, update notifications, and auth prompts all display correctly via `__blurbyStub.emit()`.

7. **Onboarding flow is solid.** Welcome screen, tour tooltips, skip option, reset, and skip-to-library all work as expected.

## Verdict

**PASS with caveats.** The app is functionally solid across all core features. The failures are concentrated in keyboard G-sequences (automation timing) and one possible Ctrl+, conflict. No blocking issues found. Ready for CT-3 to address the shortcut checklist alignment and any manual G-sequence verification.
