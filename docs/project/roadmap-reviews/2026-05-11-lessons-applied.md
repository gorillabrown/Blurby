# 2026-05-11 Lessons Applied

## Lessons Carried Forward

1. **Timing truth must fail closed.** `KOKORO-DEEPEN-2` proved the right rule: trusted word timing gets bold word highlight; missing or heuristic timing gets chunk/segment-only highlight.
2. **Evidence beats desire in model/runtime work.** `KOKORO-DEEPEN-3` showed weighted Kokoro voice formulas are not viable in the current runtime. The roadmap now defers public voice-mixing UX.
3. **Do not let optional export jump the queue.** Export depends on durable segment, cache, timing, and diagnostics contracts.
4. **Provider scope must be explicit before provider growth.** The next sprint adds capability truth before any future provider or runtime lane.
5. **Cache migrations must be non-destructive.** Legacy cache behavior should miss safely unless compatibility is proven.

## Standing Rule Reinforcement

The roadmap and queue now encode the same discipline:

- Kokoro remains the default and timing baseline.
- Qwen remains disabled.
- Model/provider expansion is not an active lane.
- New architecture work must preserve current playback behavior unless the sprint explicitly says otherwise.
