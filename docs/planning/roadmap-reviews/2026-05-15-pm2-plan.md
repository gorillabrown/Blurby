## New Sprint: TTS-CACHE-HARDEN-1

**Rationale:** The implementation review identified 9 action items that don't fit any existing sprint scope. The most critical is A8 (cache-hit timing identity parity) — `loadCachedChunk` returns only audio/words/timestamps/duration but NOT `chunkId`, `segmentId`, `timingTruth`, meaning cache hits and cache misses are observably different to downstream consumers. This blocks TTS-EVENT-SYNC-1's event-driven sync path, which needs timing truth metadata to make correct highlight decisions.
