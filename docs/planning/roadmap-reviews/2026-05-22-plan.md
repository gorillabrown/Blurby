## Sprint Shapes

### READER-PERSISTENT-ANCHOR-STEP3-REPAIR - Persistent Anchor Manual QA Repair

- **What:** Repair Page jump-back, Focus active render, Flow follow/browse-away, Narrate exact start, hard-click retarget, button styling, and reopen drift while preserving Step 2 wins.
- **Why:** Step 2 is automation-green but manual-QA-red; adapter isolation should not extract broken behavior.
- **Done when:** The 18-scenario manual QA matrix passes or any residual miss has explicit user-approved disposition.
- **Key areas:** FoliatePageView, ReaderContainer, ReaderBottomBar, reader mode hooks, persistent anchor utilities, Foliate sync, document lifecycle restore.
- **Manual gate:** Screen-interaction QA required before merge.

### READER-ISO-1A - Adapter Contracts + Current Word Anchor Service

- **What:** Add type-only mode adapter contracts plus a tested current-word anchor service.
- **Why:** Preserve exact anchors, including word `0`, after Step 3 fixes hard-selected vs last-read precedence.
- **Done when:** Adapter contract tests and anchor precedence tests pass; no runtime behavior changes.
- **Source:** Isolation spec Phase 1 plus SRL-054.

### READER-ISO-1B - Orchestrator Shell + Mode Selection Semantics

- **What:** Extract mode selection/start/pause/stop routing from `useReaderMode.ts` into an orchestrator shell.
- **Why:** Stop using shared helpers such as `startFlow(...)` to carry Narrate intent.
- **Done when:** Selecting modes never auto-starts, Play/Space starts selected mode, `C` remains global, and delayed readiness retries preserve mode intent.

### READER-ISO-1C - Focus Adapter + Passive Surface Contract Start

- **What:** Move Focus lifecycle behind the new adapter boundary and begin passive surface command typing.
- **Why:** Focus is the lowest-risk adapter migration and proves the contract before Flow/Narrate.
- **Done when:** Focus starts from exact current anchor, updates shared anchor only while active, and does not mutate Flow/Narrate runtime state.

### READER-ISO-1D - Flow Adapter + Section Handoff Restart Ownership

- **What:** Move Flow lifecycle and FlowScrollEngine ownership behind a Flow adapter.
- **Why:** Flow needs independent ownership so section restart and rolling-window behavior cannot disturb Narrate.
- **Done when:** Flow exact-start, single underline cursor, section restart, browse-away pause, and Narrate non-interference tests pass.
