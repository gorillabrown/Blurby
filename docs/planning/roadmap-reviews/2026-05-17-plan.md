# Roadmap Plan — 2026-05-17

## Finish Line: TTS Quality Confidence + Reading Experience v2

**Definition:** The TTS architecture (complete) is exercised under quality gates that catch regressions, AND the narration UX improvements that sit on top of that architecture ship to users — making Blurby's daily reading experience noticeably better.

**Exit criteria:**
1. TTS quality gates run in CI and block regressions
2. MediaSession controls work from lock screen / Bluetooth
3. Pause/resume behavior is context-aware (named-pause)
4. Narration cursor holds still during inter-word silence
5. Spoken/display word pipeline is separated for cleaner alignment
6. Library and reader UX wins ship (cards, command palette, space-bar mode)

---

## Macro Steps (5)

1. **Lock-screen & headphone controls** — MediaSession API wires narration to OS media transport
2. **Smarter pause/resume** — Named-pause state machine discriminates why playback stopped
3. **Silence-aware cursor** — Visual cursor holds during inter-word gaps instead of interpolating through silence
4. **Spoken/display separation** — Clean pipeline boundary between phoneme-alignment input and display tokens
5. **Quality gates + UX bundle** — TTS eval CI integration + library/reader polish

## Sequencing Rationale

- **NARR-MEDIA-1 first**: Smallest sprint, highest daily-use value (Bluetooth headphone users get play/pause/skip immediately). Eat the frog.
- **NARR-PAUSE-1 second**: Small, enables NARR-CURSOR-2 to make smarter decisions about cursor behavior during rate-change pauses vs. user stops.
- **NARR-CURSOR-2 third**: Uses existing `endTime` data from NARR-TIMING; pause-reason awareness lets it distinguish "hold during silence" from "user paused."
- **TTS-EVAL-3 parallel**: Independent track — can run alongside narration UX work. Exercises the existing harness against real scenarios and establishes baseline thresholds.
- **NARR-SPOKEN-1 after Stage 1a**: Medium complexity, benefits from the cleaner state after pause/cursor work stabilizes the narration hot path.
- **UX-POLISH-1 flexible**: Library/reader UX wins are independent of narration pipeline — can slot anywhere.
- **TTS-QUAL-CI-1 last**: Takes TTS-EVAL-3 results and wires them into CI. Depends on having established thresholds.
