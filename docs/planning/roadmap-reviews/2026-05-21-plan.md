# Roadmap Plan - 2026-05-21

## Revised Finish Line

**TTS Quality Confidence + Reading Experience v2** remains the finish line, but the next stage is now **Reader Runtime Solidification**.

Definition: Flow, Narrate, Focus, and Page can be changed independently without breaking each other's anchors, cursors, visual state, pacing owners, or keyboard/navigation behavior, and the test suite is clean enough for CI gating.

## Proposed Phase Structure

### Phase A - Reader Runtime Solidification

Goal: turn the Phase 0 stabilization plus GOVERNANCE-SWEEP into a clean baseline, restore broad-suite confidence, and insulate reader modes behind adapter ownership contracts.

### Phase B - Quality Gate Activation

Goal: wire quality gates and CI once broad-suite health is reliable enough to make failures meaningful.

### Phase C - Reading Experience UX Polish

Goal: resume library cards, command palette, and space-bar polish after the reader runtime no longer shifts underneath them.

## Sprint Shapes

### BASELINE-SYNC-1 - Commit Phase 0 + Governance Sweep Baseline

- **What:** Review, explicitly stage, and commit the Phase 0 runtime/test/spec/governance bundle plus GOVERNANCE-SWEEP hygiene changes while excluding unrelated local dirt.
- **Why:** Adapter work and queue resequencing must start from a named, recoverable baseline.
- **Done when:** Phase 0 files, governance sweep files, close-outs, roadmap-review artifacts, and roadmap/workbook sync are committed in intentional commits; `git status` only shows explicitly deferred dirt; focused Phase 0 tests pass; governance sweep search checks still pass.
- **Key files:** `src/components/FoliatePageView.tsx`, `tests/useReaderMode.test.ts`, `tests/phase0Stabilization.test.ts`, isolation spec, governance sweep archives/readmes/content repairs, close-outs, SpecRetro, roadmap, sprint queue workbook.
- **Commit shape:** Prefer split commits: Phase 0 runtime/test/spec, governance sweep hygiene, roadmap/sprint-queue sync.

### TEST-GREEN-1 - Broad Suite Failure Triage

- **What:** Reproduce and classify the 12 broad-suite failures reported by Phase 0.
- **Why:** `TTS-QUAL-CI-1` cannot be meaningful while unrelated broad-suite failures are waived sprint after sprint.
- **Done when:** Each failing file is classified as fix-now, quarantine/skip-with-ticket, obsolete test, or product bug; at least the default suite has a clear green path.
- **Key areas:** PDF export, FlowScrollEngine chunks, TTS cache/parity, silence cursor, CSS injection, structural checks.

### READER-ISO-1A - Adapter Contracts + Current Word Anchor Service

- **What:** Add type-only mode adapter contracts plus a tested current-word anchor service.
- **Why:** Preserve exact anchors, including word `0`, before moving runtime behavior.
- **Done when:** Adapter contract tests and anchor precedence tests pass; no runtime behavior changes.
- **Source:** `docs/planning/specs/2026-05-21-reader-mode-runtime-isolation-design.md` Phase 1.

### READER-ISO-1B - Orchestrator Shell + Mode Selection Semantics

- **What:** Extract mode selection/start/pause/stop routing from `useReaderMode.ts` into an orchestrator shell.
- **Why:** Stop using shared helpers such as `startFlow(...)` to carry Narrate intent.
- **Done when:** Selecting modes never auto-starts, Play/Space starts selected mode, `C` remains global, and delayed readiness retries preserve mode intent.
- **Source:** Isolation spec Phase 2.

### READER-ISO-1C - Focus Adapter + Passive Surface Contract Start

- **What:** Move Focus lifecycle behind the new adapter boundary and begin passive surface command typing.
- **Why:** Focus is the lowest-risk adapter migration and proves the contract before Flow/Narrate.
- **Done when:** Focus starts from exact current anchor, updates shared anchor only while active, and does not mutate Flow/Narrate runtime state.
- **Source:** Isolation spec Phase 3 and Phase 6 setup.

- `UX-POLISH-1` remains valuable, but should wait until Flow/Narrate runtime isolation no longer churns shared reader controls.
- The 6 GOVERNANCE-SWEEP deferred review items should be handled by `GOV-HUMAN-REVIEW-1` or folded into a later governance cleanup sprint.
