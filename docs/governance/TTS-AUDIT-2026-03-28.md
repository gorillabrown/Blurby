| **1** | `audioPlayer.ts:70` | `buffer.copyToChannel(new Float32Array(float32.buffer as ArrayBuffer), 0)` — when `pcmData` is a plain `number[]`, `float32` is a new Float32Array. Its `.buffer` is a fresh ArrayBuffer. But when `pcmData` is already a Float32Array, we reuse its buffer. If the original buffer is an `ArrayBuffer` from IPC (structured clone), the cast works. But if it was a `SharedArrayBuffer` (transferable), this cast is wrong and would throw silently or produce corrupt audio. | Corrupt audio or silent playback with certain IPC transfer modes. Currently safe because `Array.from(pcm)` in tts-worker.js means pcmData is always `number[]`, but fragile if worker changes to transfer Float32Array. | Always construct a fresh Float32Array from the data: `const float32 = new Float32Array(pcmData);` regardless of input type. Remove the ternary. |
| **2** | `audioPlayer.ts:38-47` | `setInterval` fires at `currentMsPerWord` intervals. If `currentMsPerWord` is fractional (e.g. 83.3ms for 12 words in 1000ms), timer drift accumulates. After 12 words, total drift can be ~5ms per word × 12 = 60ms, causing the last word callback to fire noticeably before or after audio ends. | Word highlight desyncs from audio toward end of longer chunks. | Not critical given chunks are sentence-length (5-15 words), but could improve by using `AudioContext.currentTime` based approach inside a single `requestAnimationFrame` loop instead of setInterval. ## Priority Classification

### MUST FIX (will cause user-visible bugs)

- **#5** — computeChunkPauseMs reads stale reducer state for pre-buffer check
- **#6** — cursorWordIndex not updated in stateRef after CHUNK_COMPLETE → re-speaks same chunk
- **#7 + #8** — preBufferRef not cleared on speed change → plays audio at wrong speed
- **#10** — preBufferRef not cleared on stop → stale buffer persists across sessions
- **#20** — stateRef.status not updated on pause → race with async chunk dispatch
- **#21** — stateRef.status not updated on resume → next chunk chaining may fail

- **#4** — rapid pause calls → word offset drift
- **#9** — preBufferRef not cleared on resync
- **#11** — inFlight guard silently drops chunks with no feedback
- **#16** — worker crash doesn't auto-recover
