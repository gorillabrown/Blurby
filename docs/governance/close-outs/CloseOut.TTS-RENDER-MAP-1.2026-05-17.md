## Sprint Brief

**Goal:** Replace per-word live DOM lookup on the narration highlight path with a pre-built word-index position map that is rebuilt on known render/layout triggers.  
**Result:** `WordPositionIndex` landed and is now consumed by `FoliatePageView` as the primary word-position resolver, with diagnostics for build/miss events plus graceful DOM-query fallback when index entries are stale or missing.  
**Learned:** The index path must remain fail-open to DOM fallback while section loads/reflows are in-flight; strict index-only behavior would regress continuity during transient render churn.  
**Recommend:** Dispatch `TTS-PIPELINE-1` next from clean `main`.  
**Bottom line:** TTS-RENDER-MAP-1 shipped cleanly to `main` with rebuild triggers, diagnostics instrumentation, and regression gates passing.
